## 一、单据的实体构成

### 业务对象

系统业务功能由不同的业务对象构成，核心以单据（表单）为基础。

### 单据的组成

单据 = **布局控件** + **实体字段**。
- 布局控件：仅界面展示，不存储物理数据。
- 实体字段：有展示效果，并存储到数据库物理表。

### 四种实体及其关系

| 实体类型 | 类比（PPT） | 说明 |
|---------|------------|------|
| **单据头** | 底页 | 代表整个单据 |
| **子单据头** | 形状分区 | 对单据头信息分类，一对多 |
| **单据体** | 表格 | 分录数据，多行 |
| **子单据体** | 嵌套表格 | 挂载在单据体行下，一对多 |

层次关系：
- 1个单据 = 1个单据头
- 1个单据头 = n个子单据头
- 1个单据头 = n个单据体
- 1个单据体 = n个子单据体（一般不推荐）
- 子单据体还可以挂下级子单据体（一般不推荐）

**注意**：
- 子单据头直接关联单据头（一对一），不同单据体分录之间无直接关联
- 子单据体必须和父单据体直接关联，不能与其他实体直接关联
- 通俗理解：单据头是父亲，子单据头是儿子，单据体是儿子，子单据体是孙子

## 二、Python插件中对实体数据包的操作

### 读取/更新单据数据的三种方式

1. **从界面上读取/更新** — 只能在界面类插件中使用
2. **从单据体的实体数据中读取/更新** — 最常用
3. **直接通过SQL从数据库读取/更新**

### DynamicObject 概述

实体数据包主要是 **DynamicObject** 对象（动态数据对象）。

- DynamicObject 相当于一个**有层次结构的数据字典**，通过键值对方式构成
- 可以类比为 **JSON 对象**，需要一层一层剥开去操作
- 与 JSON 不同：每个 DynamicObject 有特定类型（由单据实体结构决定），不能随意添加/删除属性

**Key 使用规则**：
- 读取子单据头、单据体、子单据体时 → 使用 **ORM 实体名**
- 读取字段时 → 使用 **绑定实体属性**

### 完整操作示例（以采购订单为例）

以下为一个完整的 Python 表单插件数据操作示例，展示从获取单据头到读取嵌套基础资料、子单据头、单据体、子单据体的全过程。

#### 0. 模板代码结构

#### 1. 获取单据头数据包

```python
billObj = this.View.Model.DataObject
# billObj 就是单据头的完整数据包，DynamicObject 类型
```

#### 2. 读取单据头字段

```python
# 单据编号（字符串）
billNo = billObj["BillNo"]

# 单据ID/内码（整数）
billId = billObj["Id"]

# 采购日期（日期类型）
```

#### 3. 读取基础资料字段（以供应商为例）

```python
# 获取供应商数据包
supplierObj = billObj["SupplierId"]
# 注意：基础资料字段取出的是 DynamicObject 类型（资料的数据包）

if supplierObj is not None:
    # 供应商编码
    supplierNum = str(supplierObj["Number"])

    # 供应商名称（多语言文本，可直接转字符串）
    supplierName = str(supplierObj["Name"])

    # 供应商ID（内码）方式1：从资料数据包读取
    supplierId = str(supplierObj["Id"])

# 供应商ID（内码）方式2：直接从单据头读取（推荐）
# key 固定格式: "资料字段属性key_Id"
supplierId = str(billObj["SupplierId_Id"])
```

#### 4. 读取基础资料嵌套的子单据头（供应商财务信息）

```python
# 供应商财务信息属于供应商的子单据头
# 取到的是 DynamicObjectCollection，固定只有1个元素
supplierFinObj = supplierObj["SupplierFinance"]

if supplierFinObj is not None and supplierFinObj.Count > 0:
    # 付款条件Id
    payConditionId = supplierFinObj[0]["PayCondition_Id"]
    # 付款条件数据包
    payConditionObj = supplierFinObj[0]["PayCondition"]
```

#### 5. 读取子单据头（表头财务信息）

```python
# 采购订单表头价税合计属于财务信息子单据头
FinObj = billObj["POOrderFinance"]
# DynamicObjectCollection 类型，固定1个元素

if FinObj is not None and FinObj.Count > 0:
    billAllAmount = FinObj[0]["BillAllAmount"]  # 浮点数
```

#### 6. 读取单据体（采购订单明细）

```python
entity = billObj["POOrderEntry"]
# DynamicObjectCollection 类型，每行一个 DynamicObject

for rObj in entity:
    # 明细ID内码（未保存时为0）
    entryId = rObj["Id"]

    # 物料数据包
    matObj = rObj["MaterialId"]

    # 物料Id 方式1
    matId = rObj["MaterialId_Id"]
    # 物料Id 方式2
    matId = matObj["Id"]

    # 物料编码
    matNum = matObj["Number"]
    # 物料名称（多语言文本）
    matName = str(matObj["Name"])

    # 采购数量（浮点数）
    Qty = rObj["Qty"]
    # 含税单价（浮点数）
    taxPrice = rObj["TaxPrice"]
```

#### 7. 读取子单据体（交货明细）

##### 方式1（直接通过实体名读取，不推荐）

```python
for rObj in entity:
    subEntity = rObj["POOrderEntryDeliPlan"]
    # DynamicObjectCollection 类型
    for subRowObj in subEntity:
        subEntryId = subRowObj["Id"]                   # 子单据体行ID
        deliveryDate = subRowObj["PlanDeliveryDate"]   # 计划交货日期
```

##### 方式2（通过元数据读取，推荐）

```python
# 需要添加引入:
# from Kingdee.BOS.Core.Metadata.EntityElement import *
# 注意获取元数据时使用标识 FEntryDeliveryPlan

subEn = this.View.BillBusinessInfo.GetEntity("FEntryDeliveryPlan")
# SubEntryEntity 类型

for rObj in entity:
    subEntity = subEn.DynamicProperty.GetValue(rObj)
    # 传入父分录的行数据包，返回子单据体 DynamicObjectCollection
    for subRowObj in subEntity:
        subEntryId = subRowObj["Id"]
        deliveryDate = subRowObj["PlanDeliveryDate"]
```

#### 8. 修改数据包

```python
# 修改单据编号
billObj["BillNo"] = "test001"

# 修改单据体字段
for rObj in entity:
    matId = 1001  # 实际要用系统中可用的物料ID

    # 根据物料Id获取物料数据包
    matFld = this.View.BillBusinessInfo.GetField("FMaterialId")
    matObj = BusinessDataServiceHelper.LoadSingle(
        this.Context, matId, matFld.RefFormDynamicObjectType
    )

    # 对资料类型字段赋值：需要同时赋值ID和数据包
    rObj["MaterialId_Id"] = matId    # 物料Id赋值
    rObj["MaterialId"] = matObj      # 物料数据包赋值
    rObj["Qty"] = 100                # 修改采购数量
    rObj["TaxPrice"] = 10            # 修改含税单价

# 修改完实体数据包后，需要刷新界面
this.View.UpdateView("FBillNo")          # 刷新单据编号字段
this.View.UpdateView("FPOOrderEntry")    # 刷新整个单据体
```

#### 9. 修改数据包注意事项

1. **只读字段不能赋值** — 例如基础资料属性字段、基础资料数据包里面的字段
2. **实体ID内码不要修改** — 由系统统一管理
3. **数据类型必须匹配** — 数量字段赋小数，日期字段赋日期，字符串字段赋字符串，DynamicObject 字段赋数据包

#### 10. 构建新的 DynamicObject（新增单据体行）

```python
# 获取新行的数据类型
newRow = DynamicObject(entity.DynamicCollectionItemPropertyType)

# 对 newRow 里面的字段逐个赋值（不会触发实体服务规则和值更新）
# newRow["字段标识"] = 值
# ...

# 添加到单据体
entity.Add(newRow)
```

### 补充说明：数据赋值操作建议配合操作服务插件使用

以上演示的实体数据包读取/修改操作在各类插件中均适用，但在实际开发中需要根据场景选择合适的插件类型：

| 场景 | 推荐插件类型 | 说明 |
|------|------------|------|
| 界面展示控制/交互（锁定、可见性、颜色等） | **表单插件 / 列表插件** | 依赖界面触发，操作 View 控件 |
| 保存/提交/审核时对数据进行赋值处理 | **操作服务插件** | 事务保护，支持批量，不依赖界面 |

**为什么推荐在操作服务插件中处理数据赋值？**

- **受事务保护** — `BeginOperationTransaction` / `EndOperationTransaction` 中的操作支持回滚
- **列表和维护界面均可触发** — 不局限于单据打开界面
- **支持批量处理** — `e.DataEntitys` 中包含了所有待处理的单据数据包
- **代码量低** — 校验逻辑可通过 BOS 配置实现，插件只需关注核心业务

**典型场景示例**（在操作服务插件中修改数据包）：

```python
def OnPreparePropertys(e):
    """预加载需要操作的字段"""
    e.FieldKeys.Add("F_VF_FKBL")      # 付款比例
    e.FieldKeys.Add("FAPPLYAMOUNTFOR") # 申请付款金额


def BeginOperationTransaction(e):
    """事务内对数据进行赋值（支持回滚）"""
    for bill in e.DataEntitys:
        payRatio = Convert.ToDecimal(bill["F_VF_FKBL"])
        if payRatio == 0:
            continue

        SrcEntryObjs = bill["FPAYAPPLYENTRY"]
        for entry in SrcEntryObjs:
            price = Convert.ToDecimal(entry["FPAYAMOUNTFOR"])
            entry["FAPPLYAMOUNTFOR"] = Math.Round(price * payRatio, 6)
```
