## 二、Python插件脚本代码构成"三部曲"

### 概述

在基于 IronPython 开发插件（例如金蝶 BOS 等平台）时，代码结构通常遵循固定的三个部分：

1. **添加类库引用**（`clr.AddReference`）
2. **引入命名空间**（`from ... import *`）
3. **定义方法/函数**（`def`）

下面详细解释每一部分的作用、语法以及注意事项。

---

### 第一部：添加类库引用（clr.AddReference）

#### 为什么需要这一步？

在 Python 插件中（特别是 IronPython），我们需要调用 .NET 平台提供的各种 DLL 类库。为了能够加载这些 DLL，必须借助 `clr` 模块（Common Language Runtime）。

#### 具体操作

1. **导入 clr 模块**
   ```python
   import clr
   ```
   - `clr` 是 pythonnet 提供的模块，让 Python 可以与 .NET 程序集交互。
   - 必须首先导入，否则无法使用 `clr.AddReference`。

2. **引用具体的 DLL 类库**
   ```python
   clr.AddReference("Kingdee.BOS.ServiceHelper")
   ```
   - 相当于 C# 插件中"添加 DLL 文件引用"。
   - 初学者理解：就像打开了一个文件夹，这个文件夹里装着很多工具（类、方法等）。
   - 只有引用了 DLL，才能进一步导入该 DLL 内部的命名空间。
   - **注意**：不执行 `clr.AddReference`，后续的 `from ... import` 会失败。

---

### 第二部：引入命名空间（from ... import *）

#### 为什么需要这一步？

引用了 DLL 之后，还需要告诉 Python 具体要用这个 DLL 中的哪个"命名空间"（可以理解为文件夹里的某个文件）。C# 中使用 `using` 关键字，Python 中使用 `from ... import`。

#### 语法

```python
from 命名空间 import *
```

- `*` 表示导入该命名空间下的所有内容（类、方法等）。
- 也可以只导入需要的特定类：`from 命名空间 import 类名`

#### 示例对照

| C# 代码 | Python 插件代码 |
|---------|----------------|
| `using Kingdee.BOS.ServiceHelper;` | `from Kingdee.BOS.ServiceHelper import *` |

- **特别注意**：Python 语句末尾不需要分号（`;`）。

#### 最佳实践

虽然 `import *` 方便，但可能导致命名冲突。建议只导入需要的类：

```python
from Kingdee.BOS.ServiceHelper import DBServiceHelper
```

---

### 第三部：定义方法/函数（def）

#### 方法的作用

- **事件方法**：插件框架会预定义一些事件，例如 `OnPreparePropertys`、`BeginOperationTransaction` 等。你需要编写这些方法的具体逻辑。
- **自定义函数**：将复杂或重复的逻辑封装成自定义函数，然后在事件方法中调用。

#### 语法基础

```python
def 函数名(参数):
    """可选的文档字符串"""
    # 函数体
    return 结果   # 可选
```

#### 示例

1. **定义事件方法**
   ```python
   def OnPreparePropertys(e):
       e.FieldKeys.Add("FBillTypeID")
   ```

2. **定义自定义函数并在事件中调用**
   ```python
   # 自定义函数
   def get_count(sql):
       from Kingdee.BOS.ServiceHelper import DBServiceHelper
       return DBServiceHelper.ExecuteScalar(sql)

   # 事件方法
   def BeginOperationTransaction(e):
       sql = "SELECT COUNT(*) FROM T_ITEM"
       total = get_count(sql)
   ```

---

### 完整示例代码

以下是一个遵循"三部曲"结构的完整 Python 插件脚本：

```python
# 1. 导入 clr 并添加类库引用
import clr
clr.AddReference("Kingdee.BOS.Core")
clr.AddReference("Kingdee.BOS.ServiceHelper")

# 2. 引入需要的命名空间
from Kingdee.BOS.Core import *
from Kingdee.BOS.ServiceHelper import DBServiceHelper

# 3. 定义自定义函数和事件方法
def execute_sql(sql):
    return DBServiceHelper.ExecuteNonQuery(sql)

def OnPreparePropertys(e):
    e.FieldKeys.Add("FBillTypeID")

def BeginOperationTransaction(e):
    for billObj in e.DataEntitys:
        billId = billObj["Id"]
```

---

### 总结对照表

| 组成部分 | 作用 | Python 语法 | 类似 C# 的概念 |
|---------|------|-------------|---------------|
| 类库引用 | 加载外部 DLL | `clr.AddReference("DLL名称")` | 添加 DLL 引用 |
| 命名空间引入 | 使用 DLL 中的类型 | `from 命名空间 import *` | `using 命名空间;` |
| 方法定义 | 实现事件或自定义逻辑 | `def 方法名(e):` | `public void 方法名(...)` |

---

### 常见错误与注意事项

- **忘记 import clr**：直接使用 `clr.AddReference` 会报错 `NameError: name 'clr' is not defined`。
- **结尾加了分号**：Python 语句不需要分号，加了虽然不报错但不规范，且容易与 C# 混淆。
- **引用 DLL 后忘记导入命名空间**：虽然引用了 DLL，但没有 `from ... import`，仍然无法直接使用其中的类。
- **事件方法名拼写错误**：插件框架通过方法名识别事件，大小写必须完全匹配（例如 `OnPreparePropertys` 不能写成 `onpreparepropertys`）。
- **混淆了 C# 和 Python 的语法习惯**：例如在 C# 中用 `{}` 表示代码块，而 Python 依靠缩进。请严格使用缩进表示层级。

## 三、Python插件中的语法注意事项

Python 属于脚本语言，一般直接在 BOS 平台中点开【注册Python脚本】编辑即可。也可以使用 VS 或其他 Python 编码工具，甚至记事本。**不要使用带格式的编辑器，例如 Word、网页在线编辑等。**

- **缩进格式**：Python 需要严格缩进对齐，尽量使用 TAB 键进行代码缩进对齐。
- **一行一码**：Python 脚本中一行只能写一句代码，例如 `a=1;b=2;` 必须写 2 行。
- **代码结束符**：由于 Python 中一行就代表一句代码，所以可以不写任何结束符。当然，也可以用 `;` 结束。
- **一级管一级**：Python 中没有大括号 `{...}` 来标记代码所属范围，如方法定义、循环体、条件判断等。完全通过缩进对齐来控制代码所属关系。如果 `if` 下面一行代码都没有，还是会报错。
- **全局变量**：Python 中没有变量声明，对于全局变量，在方法外声明，在不同方法中引用时通过 `global` 关键字进行声明。一定要先声明，再使用，否则系统会认为是方法体中的新变量。
- **对象实例化**：Python 中没有 `new` 关键字，要实例化一个对象，相当于调用构造方法，例如 `a=Student()`。
- **多行字符串常量**：由于 Python 中一行只能写一句代码，长字符串可用 `"""..."""` 标记（3 个双引号）。
- **模板类**：把 `<T>` 改成 `[T]`。例如 C# 中 `var lst = new List<string>()`，Python 中写成 `lst = List[str]()`。
- **反射代理**：C# 插件中有时会用到反射代理类。在 Python 插件中将反射代理的类直接实例化使用。例如 C# 中 `IViewService viewService1 = Kingdee.BOS.Contracts.ServiceFactory.GetService<IViewService>(this.Context)`，Python 中写成 `viewService1 = ViewService()`（需要引入 `ViewService` 类的命名空间）。

## 四、Python插件常见错误说明

### 编译报错（BOS中编辑完脚本点击确定后）

| 报错提示 | 原因 |
|---------|------|
| `Could not add reference to assembly Kingdee.BOS.App` | **此报错可以忽略**，是目前 BOS 平台的 BUG |
| `unexpected token '<newline>'` | 不该出现换行的地方出现了换行，通常是需要打 `:` 的地方没打，例如 `if` 后面、方法定义后面等 |
| `unexpected indent` | 缩进格式不对，通常是用空格敲了缩进，或者复制了网页的代码导致格式混乱，重新用 TAB 键进行缩进 |
| `unexpected token '：'`、`unexpected token '；'` | 代码中出现了非法字符，通常是误敲了中文冒号、分号等 |
| `EOL while scanning single-quoted string` | 多行字符串常量定义有误，参考上面第三点中的多行字符串常量定义即可解决 |

### 代码运行报错

| 报错提示 | 原因 |
|---------|------|
| 值"Microsoft.Scripting.Interpreter.InterpretedFrameInfo"不是"System.String"类型，不能在此泛型集合中使用 | 代码中调用的类方法中出现了异常，系统无法解析。通常是执行 SQL 报错，检查 SQL 即可；其他情况要针对具体代码分析，通常也是调用某个类的方法时传入的参数有问题 |
| `global name 'DBServiceHelper' is not defined` | "DBServiceHelper"没有定义，通常是没有引入对应的命名空间，需要添加命名空间引入 |
| `No module named Core` | 引用的命名空间有错，不存在"Core"，请检查命名空间引入写法是否有误，或者缺少添加对应类库引用 |
| `XXX is not Callable` | XXX 不是一个方法，不能直接调用，一般是出现 `XXX()` 这种写法，去掉括号试试 |
