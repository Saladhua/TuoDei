## 一、表单插件简介

表单插件是 **Web 服务层插件**，需要依赖于单据维护界面（View）才能触发。

**关键特性：**
- 单据列表**不会**触发表单插件
- 单据下推过程本身**不触发**实体服务规则、值更新事件以及表单插件
- 遇到需要后台触发表单插件、实体服务规则、值更新的场景，可在后台构建单据的界面对象 View 来触发使用

**适用范围：**
- 单据
- 动态表单
- 基础资料

在 C# 插件开发中，表单插件的基类是 `AbstractBillPlugIn`，需要建一个派生类来实现。而在开发 Python 插件时，在注册表单插件的位置注册 Python 脚本，系统就默认是表单插件了，不需要过多考虑基类，需要更关注需要实现的**事件方法**。

## 二、表单插件中的重要成员介绍

插件基类提供了许多重要的成员供开发时使用，在插件中可以通过 **this** 关键字来调用。

### Context 上下文对象

```python
this.Context.UserId                               # 当前用户ID
this.Context.UserName                             # 当前用户名称
this.Context.UserPhone                            # 当前用户手机号
this.Context.CurrentOrganizationInfo.ID           # 当前组织Id
this.Context.CurrentOrganizationInfo.Name         # 当前组织名称
this.Context.DBId                                 # 当前数据中心Id
this.Context.DataCenterNumber                     # 当前数据中心编码
this.Context.DataCenterName                       # 当前数据中心名称
this.Context.IpAddress                            # 客户端本机网络信息:IP、MAC等
```

### View 视图层对象

可理解为当前界面，界面类插件特有，服务类的插件没有这个。

#### View 的常用属性

```python
this.View.BillBusinessInfo                              # 单据的业务逻辑元数据
this.View.BillBusinessInfo.GetBillNoField().FieldName   # 单据编号字段名
this.View.BillBusinessInfo.GetBillStatusField().FieldName  # 单据状态字段名
this.View.BillBusinessInfo.GetBillTypeField().FieldName # 单据类型字段名
this.View.BillBusinessInfo.GetForm().Id                 # 单据FormId
this.View.BillBusinessInfo.MainOrgField.FieldName       # 主业务组织字段名
this.View.BillBusinessInfo.GetEntity("FBillHead").TableName  # 单据头表名
this.View.BillBusinessInfo.GetEntity("实体标识").TableName   # 实体主表名
this.View.BillBusinessInfo.GetEntity("实体标识").SplitTables  # 实体所有拆分表

this.View.OpenParameter                                # 表单入口参数
this.View.OpenParameter.Status                         # 当前界面状态:0新增,1查看,2修改
this.View.OpenParameter.GetCustomParameter("参数标识")  # 获取单据打开传入的参数

this.View.ParentFormView                               # 获取父页面的View
this.View.ParentFormView.BillBusinessInfo.GetForm().Id # 父页面的FormId,用来判断单据从哪里进来
```

#### View 的常用方法

```python
# 标题操作
this.View.GetFormTitle()                             # 获取单据标题
this.View.SetFormTitle(LocaleValue("新标题"))          # 修改单据标题

# 操作调用
this.View.GetFormOperation("操作代码")                 # 获取单据的一个操作实例对象
this.View.InvokeFormOperation("操作代码")             # 触发单据的某个操作:保存、提交、审核、关闭等

# 控件操作
this.View.GetControl[控件类]("控件标识")                # 获取单据上的控件:按钮、菜单等,可设置可见性、锁定性等

# 值更新
this.View.InvokeFieldUpdateService("字段标识", 行号)   # 触发字段值更新,单据头字段行号填0

# 触发实体服务规则
obj = BOSActionExecuteContext(this.View)
this.View.RuleContainer.RaiseDataChanged("字段标识", 字段所在实体行数据包, obj)

# 提示消息
this.View.ShowMessage("绿色背景提示信息")               # 显示正常提示信息
this.View.ShowWarnningMessage("黄色背景提示信息")       # 显示警告提示信息
this.View.ShowErrMessage("红色背景提示信息")            # 显示错误提示信息

# 刷新界面
this.View.UpdateView("字段标识/单据体标识")             # 刷新界面数据
```

### Model 单据数据模型

单据的实体数据包获取从这里开始，结合第 3 篇所讲的实体数据包。

```python
this.View.Model                                       # 单据数据模型
this.View.Model.DataObject                           # 单据的完整数据包,相当于单据头实体数据包

# 行操作
this.View.Model.GetEntryCurrentRowIndex("单据体标识")  # 获取单据体当前焦点行号
this.View.Model.GetEntryRowCount("单据体标识")         # 获取单据体行数
this.View.Model.CreateNewEntryRow("单据体标识")        # 为单据体新增一行
this.View.Model.BatchCreateNewEntryRow("单据体标识", x) # 批量为单据体新增x行
this.View.Model.InsertEntryRow("单据体标识", i)        # 在第i行前插入1行
this.View.Model.DeleteEntryRow("单据体标识", i)        # 删除第i行
this.View.Model.DeleteEntryData("单据体标识")          # 清空整个单据体数据

# 取值和赋值
this.View.Model.GetValue("字段标识", x)                # 获取第x行某字段的值,单据头不传x
this.View.Model.SetValue("字段标识", "字段值", x)       # 更新第x行某字段的值,单据头不传x
this.View.Model.SetItemValueByID("字段标识", "内码id", x)   # 用资料内码id更新第x行资料字段的值
this.View.Model.SetItemValueByNumber("字段标识", "编码", x) # 用资料编码更新第x行资料字段的值
```

**注意：** 除了第 3 篇讲的实体数据包操作之外，在表单插件中还有另外的方法对单据数据进行操作。表单插件中的这些方法使用的都是**标识**，而不是**绑定实体属性名**。实体数据包操作同样适用且在表单插件中用的也很多。

## 四、表单插件常用事件介绍

### 基本事件执行顺序

插件中的事件在单据运行过程中产生，有固定执行顺序，且在不同阶段才能获取到特定的数据。

| 单据新增界面打开 | 单据修改/查看打开 | 事件描述 |
|:---:|:---:|---------|
| PreOpenForm | PreOpenForm | 准备单据打开 |
| OnInitializeService | OnInitializeService | 加载单据插件等，可动态调整插件执行顺序 |
| OnSetBusinessInfo | OnSetBusinessInfo | 绑定单据元数据 |
| OnSetLayoutInfo | OnSetLayoutInfo | 设置单据界面显示信息，如布局等 |
| OnCreateDataBinder | OnCreateDataBinder | 创建数据源绑定对象 |
| OnInitialize | OnInitialize | 单据初始化完毕，可获取一些控件备用 |
| OnBillInitialize | OnBillInitialize | 单据初始化完毕 |
| CreateNewData | — | 构建新单据初始实体数据，**新增才会触发** |
| AfterCreateNewData | — | 新单据初始实体数据构建完毕，在此可修改单据默认数据 |
| AfterCreateModelData | — | 新单据实体数据构建到 Model |
| — | LoadData | 开始加载单据实体数据 |
| — | AfterLoadData | 加载完单据数据 |
| OnLoad | OnLoad | |
| BeforeBindData | BeforeBindData | 开始绑定实体数据到界面控件 |
| AfterBindData | AfterBindData | 绑定实体数据到界面控件完毕，**尽量不要在此事件中修改数据，仅修改控件状态** |

### Python 插件中如何使用事件

1. 事件就是定义好的方法，使用 `def` 关键字，后面跟上方法名称、参数列表即可，表单插件的事件一般没有返回值
2. 事件方法参数：在参数中能获取到当前事件应提供的信息，表单插件事件一般只有 1 个参数，用 `e` 即可
3. 示例：`def AfterCreateModelData(e):` — 注意最后有英文冒号 `:` 结尾

### 常用事件用法介绍

以下列出的事件中都可以获取到单据数据包。

#### 1. AfterCreateModelData

单据新增界面打开时触发，可以获取到单据初始化之后的数据。通常用于对单据新增进行默认值填充。

```python
def AfterCreateModelData(e):
    billObj = this.Model.DataObject
    this.View.Model.SetValue("FDate", "2022-7-1")
```

#### 2. AfterBindData

界面数据绑定完毕后触发，通常在这里对字段控件状态进行设置，如设置颜色、锁定性/可见性、动态枚举选项等。

```python
def AfterBindData(e):
    this.View.GetControl("FDate").Enabled = False
    # this.View.GetControl("控件标识").Visible = False
```

#### 3. BarItemClick

单据头菜单点击事件。一定要加判断，只监听需要的菜单。多个菜单使用 `if/elif` 结构（Python 中没有 Switch）。

```python
def BarItemClick(e):
    key = e.BarItemKey.ToUpperInvariant()
    if key == "TESTBTN1":
        msg = ("菜单[{0}]点击事件捕捉到了，可以继续菜单的功能啦！").format(key)
        this.View.ShowMessage(msg)
    elif key == "TESTBTN2":
        msg = ("菜单[{0}]点击事件捕捉到了，可以继续菜单的功能啦！").format(key)
        this.View.ShowMessage(msg)
```

#### 4. EntryBarItemClick

单据体菜单点击事件，与 `BarItemClick` 类似，但监听的是单据体菜单栏。

```python
def EntryBarItemClick(e):
    key = e.BarItemKey.ToUpperInvariant()
    if key == "entryTESTBTN1":
        msg = ("单据体菜单[{0}]点击事件捕捉到了，可以继续菜单的功能啦！").format(key)
        this.View.ShowMessage(msg)
```

#### 5. ButtonClick

按钮控件/超链接控件点击事件。

```python
def ButtonClick(e):
    BtnKey = e.Key.ToUpperInvariant()
    if BtnKey == "F_ora_LINK":
        msg = ("按钮[{0}]点击事件捕捉到了，可以继续菜单的功能啦！").format(BtnKey)
        this.View.ShowMessage(msg)
```

#### 6. EntityRowDoubleClick

单据体行双击事件，使用时一定要判断单据体。

```python
def EntityRowDoubleClick(e):
    entityKey = e.Key.ToUpperInvariant()
    row = e.Row
    fldKey = e.ColKey.ToUpperInvariant()
    msg = ("单据体[{0}]第[{1}]行,字段[{2}]双击啦！").format(entityKey, row, fldKey)
    this.View.ShowMessage(msg)
```

#### 7. EntityRowClick

单据体行点击事件（单行选择事件），使用时一定要判断单据体。

```python
def EntityRowClick(e):
    entityKey = e.Key.ToUpperInvariant()
    row = e.Row
    msg = ("单据体[{0}]第[{1}]行被选中啦！").format(entityKey, row)
    this.View.ShowMessage(msg)
```

#### 8. AfterDeleteRow

单据体行删除事件，可以在此获取被删除的行数据。

```python
def AfterDeleteRow(e):
    entityKey = e.EntityKey.ToUpperInvariant()
    row = e.Row
    deletRowObj = e.DataEntity
    msg = ("单据体[{0}]第[{1}]行被删除啦！").format(entityKey, row)
    this.View.ShowMessage(msg)
```

#### 9. DataChanged

值更新事件。当 BOS 的值更新与实体服务配置实现不了想要的功能时使用。字段在 BOS 中勾选【即时触发值更新】才会触发插件值更新事件。

```python
def DataChanged(e):
    fldKey = e.Field.Key.ToUpperInvariant()
    if fldKey == "ABC":
        return
    row = e.Row
    oldValue = e.OldValue
    newValue = e.NewValue
    msg = ("第[{0}]行字段[{1}]从[{2}]更新成了[{3}]").format(row, fldKey, oldValue, newValue)
    this.View.ShowMessage(msg)
```

#### 10. BeforeF7Select

资料字段选择前事件（放大镜点击弹出选择列表前触发）。

用途：
1. **最常用**：对弹出的选择列表进行动态过滤，控制数据选择范围
2. 控制资料字段录入顺序，例如没选供应商就不能选物料
3. 非资料类型字段启用编辑按钮（3 个点...），点击时触发

```python
def BeforeF7Select(e):
    fldKey = e.FieldKey.ToUpperInvariant()
    baseFormId = e.FormId
    row = e.Row

    if fldKey == "FMATERIALID":
        supObj = this.View.Model.GetValue("FSupplierId")
        if supObj is None:
            this.View.ShowWarnningMessage("请先选择供应商")
            e.Cancel = True
            return

        e.IsShowUsed = False
        e.IsShowApproved = False
        myFilterStr = (" '{0}'='VEN00001' ").format(supObj["Number"])
        e.ListFilterParameter.Filter = myFilterStr
```

#### 11. BeforeSetItemValueByNumber

通过编码给资料字段赋值前触发。对 `BeforeF7Select` 的补充——`BeforeF7Select` 控制界面录入选择列表，但通过插件赋值不会触发 `BeforeF7Select`，而是触发此事件。

```python
def BeforeSetItemValueByNumber(e):
    fldKey = e.BaseDataField.Key.ToUpperInvariant()
    Num = e.CurrentSetNumber
    row = e.Row
    e.IsShowUsed = False
    e.IsShowApproved = False
    e.Filter = ""
```

#### 12. BeforeDoOperation / AfterDoOperation

单据界面执行单据操作前后触发。

```python
def BeforeDoOperation(e):
    opCode = e.Operation.FormOperation.Operation.ToUpperInvariant()
    if opCode == "SAVE":
        e.Cancel = True
        this.View.ShowWarnningMessage("取消保存！")


def AfterDoOperation(e):
    opCode = e.Operation.Operation.ToUpperInvariant()
    this.View.ShowMessage(str(opCode))
```
